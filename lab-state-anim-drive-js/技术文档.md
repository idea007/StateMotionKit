# State Anim Drive 技术文档

## 目标

`lab-state-anim-drive-js` 是一个浏览器侧 State Anim 驱动实验页，用于验证编辑器导出的 state anim 模板是否能在普通 Web 页面中独立运行。

当前实现是自包含版本：主要运行时能力已复制/整理到 `app.js`，路径形变能力由同目录的 `pathMorphRuntime.js` 提供，不直接依赖 `/src/editor/*`，也不依赖旧版 `vendor/` 副本。因此整个 `lab-state-anim-drive-js` 目录可以拷贝到其他 Web 项目中运行。

后续如果主工程 runtime、path morph 或模板结构有语义更新，需要同步到这个 lab 驱动。

## 文件入口

- 页面入口：`lab-state-anim-drive-js/index.html`
- 驱动逻辑：`lab-state-anim-drive-js/app.js`
- 路径形变运行时：`lab-state-anim-drive-js/pathMorphRuntime.js`
- 样式：`lab-state-anim-drive-js/styles.css`

`pathMorphRuntime.js` 是从主工程 `src/editor/pathMorph.ts` 打包出来的浏览器 ESM 文件，原则上不要手工改。需要同步主工程 path morph 时，重新生成该文件。

建议通过任意静态 Web server 访问，例如：

```bash
cd lab-state-anim-drive-js
python3 -m http.server 5211
```

然后打开：

```text
http://localhost:5211/
```

如果端口被占用，可替换为其他可用端口。

## 输入资源

页面支持选择 state anim descriptor JSON，或导出的 store 格式 zip。

descriptor 需要包含：

- `variants[]`
- `faceParts[]`
- `variants[].compiledUrl` 或内联 `variants[].motion`
- 可选 `stateMachine`

当 `stateMachine` 缺失或为 `true` 时，lab 会基于 `variants` 自动生成一个单层 fallback 状态机，方便旧资源回放。

## 运行链路

1. 读取 descriptor 或 zip。
2. 解析每个 variant 对应的 `.state.compiled.json` 或内联 compiled motion。
3. 将 compiled motion 转换成 lab 内部使用的 `AnimationDocument`：
   - `variants[]` 转成 `AnimationClip[]`。
   - `fps / frames` 转成 clip timeline。
   - `layerDefs[].motionKey` 转成 node id / track id。
   - `layers[motionKey]` 转成 track channels。
   - `layerDefs.kind = group / frame` 分别转成普通 group / frame node。
   - `layerDefs.bounds` 用作容器基础尺寸，`layerDefs.frame` 保留 frame 的填充、描边、圆角和 `clipContent`。
   - `layerDefs.parentMotionKey` 转成 node 层级关系，父级支持 group 和 frame。
   - vector path 会参考主工程导入逻辑做 bounds offset 归一化：path 数据平移到节点本地原点，position keyframes 同步补偿 offset，避免跨 clip path morph 混用原始 SVG 坐标。
   - `templateResource.faceParts` 优先按 node 保存到 `facePartsByNodeId`，避免不同 motion 的局部 part index 错配。
4. 归一化 `stateMachine`：
   - state 的 `clipId / clipRef / stateId / name` 映射到实际 clip。
   - param / transition 做兼容归一化。
   - 保留 `layerRole / control / priority / exitTime / onComplete / consumeConditions` 等配置。
5. 调用 `createStateMachineRuntime()` / `advanceStateMachine()` 推进状态机。
6. 将 `AnimationEvaluatedState.nodeOverrides` 渲染到 SVG。

## Runtime 语义

lab 内置 runtime 的目标是和主工程 State Machine 语义保持一致：

- 每个 layer 维护独立的 `currentStateId / stateFrame / transition`。
- `stateFrame` 按 clip fps 转换后的 `deltaFrames` 推进，支持小数帧。
- transition 选择支持 `any`、from state、条件、优先级。
- trigger 条件可被消费。
- 支持 `exitTime` 和 `onComplete`。
- 非循环 state 到达结束帧后可触发 onComplete transition。
- `effect.targetFramePolicy` 支持 `first-frame` 和 `current-frame`。
- 播放时使用 `requestAnimationFrame`，通过当前 active clip fps 把真实时间转换成动画帧。

页面中的 Step 按钮直接推进 1 帧；Play/Pause 使用 RAF 连续推进。

## Clip 采样

`evaluateClipAtFrame()` 负责将 clip tracks 采样为 node overrides：

- 数值属性：`position / scale / rotation / opacity / revealPath` 使用 easing 插值。
- 颜色属性：`fill / stroke` 使用 RGB 插值。
- `path` 属性：clip 内关键帧之间使用 `interpolatePathDataResult()` 做连续路径插值。
- 其它离散属性保留离散采样。
- 静态导出的默认关键帧会在归一化阶段跳过，避免导入后产生大量 0 帧噪声轨道。

支持的 easing 包括内置曲线、legacy alias、`cubic-bezier` 对象，以及简单的 bounce preset。

## Transition 过渡

State transition 的可见过渡由 `blendEvaluatedStates()` 完成：

- 不同 clip 中的图层按 `transitionMatchKey / name / id` 匹配。
- 匹配成功的图层会对 transform、opacity、fill、stroke、path、revealPath 做插值。
- from-only 图层淡出，to-only 图层淡入。
- `effect.mode = cut` 时保持 from 画面直到 transition 结束。
- 其它 effect mode 在 SVG 预览中按属性过渡展示。
- transition 开始时会生成并冻结 path morph plan，渲染每一帧复用同一套锚点/候选策略；这与主工程 State Machine runtime 保持一致，避免同一次过渡中候选策略随帧跳变。
- path morph 前的 vector path 坐标会先按导入时记录的 path bounds offset 归一化；这点和主工程 Pixi 预览一致，避免形状在过渡中被原始 path 坐标拉偏。

说明：`pixel-match` 是主工程面向栅格图像的过渡效果。当前 lab 使用 SVG 渲染，不生成像素匹配贴图；遇到这类效果会以属性过渡/淡入淡出方式展示。

## Control Layer

lab 支持 State Machine 的 control layer 投影：

- layer 为 `control` 或配置了 `control` 时，会作为控制层处理。
- control layer 自身不作为最终 motion 画面直接显示。
- control layer 的 overrides 会投射到目标 motion layer。
- 支持 `targetLayerId` 和 `targetScope`。
- 支持投影属性：
  - `transform`
  - `opacity`
  - `fill`
  - `stroke`
  - `path`
  - `revealPath`
- 投影匹配同样基于 `transitionMatchKey / name / id`。
- runtime 面板会输出 `stateMachineControlProjections`，方便排查投影目标和命中情况。

为了让 active state 的图层隔离更接近主工程预览，lab 推进时使用 `visibilityIsolation: opacity`，未参与当前 active clip 的图层通过 opacity 隐藏。

## SVG 渲染策略

SVG 渲染器只负责 lab 预览，不替代编辑器 Pixi 渲染器。

当前渲染规则：

- 画布 viewBox 以 artboard 中心为原点，尺寸取 descriptor `boardBounds` 或 compiled motion `base`。
- 由 `facePartsByNodeId` 优先创建 SVG path，缺失时回退 descriptor 全局 `faceParts`。
- group / frame / vector 按 node 层级挂载到 SVG。
- frame 会渲染自身 fill / stroke；`clipContent` 开启时会生成 SVG clipPath 裁剪子节点。
- 每帧应用 node override：
  - `position`
  - `scale`
  - `rotation`
  - `opacity`
  - `fill`
  - `stroke`
  - `path`
- stroke 只有在 stroke 存在且 `strokeWidth > 0` 时显示。
- 播放中 runtime/debug 面板降频刷新，避免每帧 JSON stringify 和 DOM 重绘影响画面流畅度；SVG 画面本身仍按 RAF 渲染。

## 一致性边界

必须尽量和主工程一致：

- state / transition 选择。
- `stateFrame` 推进。
- trigger 消费。
- `exitTime / onComplete / consumeConditions`。
- control layer 投影。
- `nodeOverrides` 的 transform / opacity / fill / stroke / path / revealPath 求值。
- path morph 的关键语义。

允许存在差异：

- SVG 与 Pixi 的抗锯齿、描边、透明合成、mask、复杂路径渲染细节。
- lab 为了预览创建的临时 node 尺寸和层级调试信息。
- `pixel-match` 等需要渲染贴图或像素级处理的效果。

如果要做到像素级对齐，应新增 Pixi/WebGL 运行页，而不是继续扩大 SVG renderer 的职责。

## 同步主工程的建议

lab 要保持可独立拷贝运行，所以不能直接 import 主工程源码。同步策略：

1. 主工程 State Machine runtime 有语义更新时，把必要逻辑同步到 `app.js`。
2. 主工程 path morph 有更新时，重新生成 `pathMorphRuntime.js`。
3. 修改后用同一个 state anim 模板分别在主工程预览和 lab 页面中验证：
   - 状态切换是否一致。
   - control layer 是否命中相同图层。
   - path morph 是否有翻转、跳变。
   - 主题/颜色投影是否一致。

后续可以补充自动化对比：

- 给定 descriptor + compiled motion，测试 `buildRuntimePackage()` 输出稳定的 `AnimationDocument`。
- 同一 state anim 在编辑器 document 与 lab document 上推进相同帧序列，比较 `runtime` 和 `nodeOverrides`。
- 为导出 zip 增加更多 fixture，覆盖 group、control layer、path morph、主题切换。

## 已知限制

- zip 目前主要面向导出的 store 格式。
- SVG 渲染用于 Web 驱动验证，不保证与 Pixi 编辑器像素级一致。
- 本地 JSON 如果引用相对 `compiledUrl`，浏览器无法自动读取相邻磁盘文件；这类资源应通过 dev server URL 或 zip 加载。
- 复杂 mask、滤镜、贴图过渡等 Pixi 能力没有在 SVG renderer 中完整实现。

## 验收

基础验收：

- 外部 descriptor JSON 或导出的 store 格式 zip 可加载。
- SVG 中有有效 path 输出。
- `runtime.source` 为 `state-machine`。
- 参数切换后状态机能推进。
- 主题/control layer 切换能投影到目标 motion layer。
- path morph transition 没有明显翻转。
- 播放时 debug 面板不应明显拖慢动画。
- 页面无 console error。
- `node --check lab-state-anim-drive-js/app.js` 通过。
- `git diff --check` 通过。
- `npm run build` 通过。

runtime 面板中的 `unsupported` 直接透出 runtime warning。某些 transition 可能出现 path fallback warning，这表示 runtime 采用了降级求值策略，不等同于页面运行错误。
